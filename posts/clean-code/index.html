意味のある名前#

  1. 意図が明確な名前にする
  2. 意味のある対比を行う
  3. 発音可能な名前を使用する
  4. 検索可能な名前を用いる
  5. マジックナンバーは定数に
  6. メンタルマッピングを避ける
  7. 抽象レベルに適切な名前を選ぶ
  8. 広いスコープには長い名前を
  9. エンコーディングを避ける
  10. 名前で副作用を示すべき

関数#

  1. 小さいこと！
  2. 1つのことを行う
  3. 内容をよく表す名前を使う
  4. 引数は少なく
  5. フラグ引数は使わず、メソッドを分割する
  6. 副作用をさける
  7. try/catchブロックの分離
  8. DRY(Don’t Repeat Yourself)
  9. 死んだ関数は削除すべき

コメント#

「ダメなコードをコメントで取り繕ってはいけない。書き直すのだ」

  1. コメントで、ダメなコードを取り繕うことはできない
  2. 自分自身をコードで説明する
  3. 意図の説明
  4. 結果に対する警告

書式化#

  1. 縦方向の書式化
  2. 垂直概念分離性
  3. 垂直密度
  4. 垂直距離
  5. 水平分離性と密度

テスト#

  1. 1つのテストに1つのアサート
  2. 1つのテストでは1つの概念を扱う
  3. カバレッジツールを使用する
  4. ささいなテストを省略しない
  5. 無視することを指定されたテストは、あいまいさへの問いかけである
  6. 境界条件テストには注意払う

  8. 失敗パターンは何かを語る
